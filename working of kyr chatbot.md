
**1. User Interaction (Frontend - React + TypeScript + Tailwind CSS):**

- **Chat UI:** The user interacts with a visually appealing and user-friendly chat interface built with React and styled with Tailwind CSS. This interface likely has:
    - An input field where the user can type their questions or describe their legal scenarios.
    - A display area to show the conversation history, including the user's questions and the chatbot's responses.
    - Visual cues like avatars and typing animations to enhance the user experience.

**2. Sending the Query (Frontend to Backend):**

- When the user submits their question (e.g., "What are my rights if I am arrested?", "Some men are harassing me on the street."), the React frontend sends this query as a text request to your FastAPI backend. This likely happens via an HTTP POST request to the `/ask` endpoint you defined.

**3. Processing the Query (Backend - FastAPI + Python):**

- **Receiving the Query:** Your FastAPI backend receives the user's text-based question.
- **Information Retrieval (FAISS + Sentence Transformers):**
    - **Embedding:** The backend uses the `sentence-transformers` library and a pre-trained model (`all-MiniLM-L6-v`) to convert the user's question into a numerical vector representation (an embedding). This vector captures the semantic meaning of the question.
    - **Similarity Search:** This vector is then used to search your pre-built **FAISS index**. The FAISS index contains vector representations of the text content from the Indian Constitution (likely broken down into articles, parts, or sections).
    - **Retrieving Relevant Context:** FAISS efficiently finds the top 'k' (in your code, `k=3`) most similar vectors in the index to the user's query vector. These similar vectors correspond to the sections of the Constitution that are most semantically related to the user's question.
    - **Accessing Metadata:** Along with the vector indices, FAISS also provides the corresponding indices in your `metadata`. This `metadata` is a Python list (loaded from `index_metadata.pkl`) where each item likely contains information about a specific section of the Constitution, such as the article number, title, and the actual text content. The backend retrieves the text content of these top matching sections.

**4. Generating the Answer (Gemini 2.0 Flash):**

- **Prompt Construction:** The FastAPI backend constructs a carefully designed prompt that is sent to the **Gemini 2.0 Flash** large language model. This prompt includes:
    - A clear instruction to act as a responsible legal assistant focused on the Indian Constitution and legal rights.
    - Specific guidelines on how to answer questions about rights, handle real-life scenarios, and what to do if a question is outside the scope.
    - The **retrieved context** from the Constitution (the top matching sections from FAISS).
    - The **user's original question**.
- **Generating the Response:** The FastAPI backend sends this prompt to the Gemini 2.0 Flash API (using the `google-generativeai` library and your `GEMINI_API_KEY`). Gemini 2.0 Flash processes the prompt, using its understanding of language and the provided context to generate a conversational and informative answer to the user's question. It's also instructed to cite relevant articles if possible.
- **Receiving the Answer:** The FastAPI backend receives the generated text response from the Gemini 2.0 Flash API.

**5. Sending the Response (Backend to Frontend):**

- The FastAPI backend constructs a JSON response containing:
    - The `answer` generated by Gemini 2.0 Flash.
    - Information about the most relevant article found (article number, title, and the source content).
- This JSON response is sent back to the React frontend.

**6. Displaying the Answer (Frontend - React + TypeScript + Tailwind CSS):**

- The React frontend receives the JSON response from the backend.
- It then dynamically updates the chat UI to display the chatbot's answer to the user, along with the source article information. The styling provided by Tailwind CSS ensures a visually appealing presentation.

**In essence, KYR Chatbot works by:**

- **Understanding the User:** Using embeddings to grasp the meaning of the user's legal question.
- **Finding Relevant Information:** Quickly retrieving the most relevant parts of the Indian Constitution using efficient vector search (FAISS).
- **Reasoning and Generating Answers:** Leveraging the power of Gemini 2.0 Flash to understand the context and the user's question and generate a natural-sounding, legally-informed answer, citing sources.
- **Presenting Information Clearly:** Providing a user-friendly chat interface to make the information accessible.

The key innovation lies in combining the speed and efficiency of vector search for retrieving specific legal information with the natural language understanding and generation capabilities of a modern LLM like Gemini 2.0 Flash. This allows the chatbot to provide accurate, contextually relevant, and easy-to-understand answers to a wide range of legal queries related to the Indian Constitution.
